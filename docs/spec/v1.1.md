# Agent A: Storage Layer

## Prerequisite: Add `skip_errors` to `session_from_jsonl`

**Why:** `session_from_jsonl` (`session.py:1271`) raises `SerializationError` on invalid
JSON (line 1298). But `StoredSession.load()` needs to skip corrupt lines for crash safety.
The current stub in `storage.py` works around this by pre-validating JSON lines before
passing them to `session_from_jsonl`, which is redundant parsing.

**Change:** Add a `skip_errors: bool = False` parameter to `session_from_jsonl`. When True,
invalid JSON lines are silently skipped instead of raising. Existing callers are unaffected
(default is False).

```python
# session.py — updated signature
def session_from_jsonl(lines: Iterable[str], *, skip_errors: bool = False) -> Session:
    ...
    for line in lines:
        line = line.strip()
        if not line:
            continue
        try:
            data = json.loads(line)
        except json.JSONDecodeError as exc:
            if skip_errors:
                continue
            raise SerializationError(f"Invalid JSON line: {exc}") from exc

        event = event_from_dict(data)
        events.append(event)
        ...

    if session_id is None or created_at is None:
        if skip_errors and not events:
            # Return empty session for completely empty/corrupt files
            raise SerializationError("JSONL stream missing SessionStarted event")
        raise SerializationError("JSONL stream missing SessionStarted event")
    ...
```

Add a test to the existing serialization tests:

```python
# tests/test_session_serialization.py — add this test
def test_session_from_jsonl_skip_errors():
    s = new_session("/tmp", "realtalk")
    good_lines = list(session_to_jsonl(s))
    bad_lines = good_lines + ['{"incomplete":']
    # Default: raises
    with pytest.raises(SerializationError):
        session_from_jsonl(bad_lines)
    # skip_errors: succeeds

```

## Design Decisions (incorporating eng review findings)

### Human-readable workspace fingerprints

**Review finding #3.** The current stub uses raw `sha256[:16]` which produces opaque
directory names like `a3f2c8b1e0d94f72/`. Bad for debugging and training export.

**Change:** Use `{dirname}-{hash[:8]}` format.

```
~/.realtalk/sessions/
  game-session-a3f2c8b1/    <- human-readable
    session.jsonl
    session.jsonl.1
    session.jsonl.2
~/.realtalk/archive/
  game-session-a3f2c8b1/    <- training data (never auto-deleted)
    20260412-143000.jsonl
```

```python
def workspace_fingerprint(cwd: Path) -> str:
    """Return '{dirname}-{hash[:8]}' for human-readable session directories."""
    resolved = cwd.resolve()
    dirname = resolved.name or "root"
    # Sanitize dirname: lowercase, replace non-alnum with hyphens
    safe = "".join(c if c.isalnum() else "-" for c in dirname.lower()).strip("-")
    safe = safe[:32]  # cap length
    hash_prefix = hashlib.sha256(resolved.as_posix().encode()).hexdigest()[:8]
    return f"{safe}-{hash_prefix}"
```

### Append-only writes are crash-safe

Each call to `append()` opens the file in append mode, writes one JSON line + `\n`,
and calls `flush()`. A crash mid-write corrupts at most the final incomplete line,
which `load()` skips via `session_from_jsonl(skip_errors=True)`.

### SessionStarted header in every rotated file

**Review finding #2.** After rotation, new files won't have a `SessionStarted` event.
This breaks `session_from_jsonl` (which requires it) and makes archived files useless
for training export.

**Change:** During rotation, after renaming the active file to `.1`, write the
`SessionStarted` header event as the first line of the new empty `session.jsonl`.
The header is read from the first line of the oldest existing rotated file.

Every file on disk is self-describing. Archived files can be loaded independently for
training export.

### Archive-before-rotate (training data preservation)

**Design doc, Approach B.** The game IS the data flywheel. Rotation that deletes old
events loses training data, which breaks the whole premise.

Before rotation deletes `session.jsonl.{keep}`, copy it to
`~/.realtalk/archive/{fingerprint}/{timestamp}.jsonl`. Training exports read from
archive + active. No data loss.

### Atomic rotation (crash safety)

**Review finding, critical failure mode.** The naive rename chain
(`path -> path.1`, then create new `path`) is not atomic. A crash between steps
could leave inconsistent state.

**Change:** Use a write-then-rename pattern for the new file:
1. Archive `.{keep}` if it exists
2. Read `SessionStarted` from the first line of the oldest file
3. Cascade rename: `.{keep-1}` -> `.{keep}`, ..., `.1` -> `.2`, `path` -> `.1`
4. Write `SessionStarted` header to a temp file, then rename temp to `path`

The rename step (3) is still multi-step, but step 4 ensures the new active file
always exists and always has the header. On load, if `.1` and the active file both
contain the same events (crash between step 3's rename and step 4), the duplicate
`SessionStarted` at the start of `.1` is harmless since `session_from_jsonl` just
reads the last `SessionStarted` seen.

### Rotation threshold via constructor, not append parameter

**Design doc resolution.** `StoredSession` accepts an optional `max_bytes` constructor
argument (default 256KB). Tests construct `StoredSession(path, max_bytes=100)` to trigger
rotation without writing 256KB. The `append()` method has no `_max_bytes` parameter.

**Change from current stub:** Remove `_max_bytes` from `append()`, add `max_bytes` to
`__init__`.

### Key implementation details

**`StoredSession` constructor:** Takes `path`, `max_bytes` (default 256KB), and `archive_root` (default None). The `archive_root` param connects the `StoredSession` to the `SessionStore`'s archive directory. When `archive_root` is None, archival is disabled and oldest rotated files are simply deleted.

**`rotate_if_needed` flow:**
1. Check file size against `max_bytes`
2. Read `SessionStarted` header from oldest available file
3. If `.{keep}` exists and `archive_root` is set, copy to `{archive_root}/{fingerprint}/{timestamp}.jsonl`
4. Delete `.{keep}`
5. Cascade rename: `.{keep-1}` -> `.{keep}`, ..., `path` -> `.1`
6. Write `SessionStarted` header to temp file, atomic rename to `path`

**`load()` flow:** Collects lines from `.3`, `.2`, `.1`, then active file (oldest to newest). Passes all lines to `session_from_jsonl(lines, skip_errors=True)`. Single-pass parsing, no double validation.
