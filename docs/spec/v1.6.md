# Realtalk — Build Spec v1.6: Tool Layer (Layer 5)

> Builds on [v1.4](./v1.4.md) (conversation runtime).
> Layers 0–3 deliver session, storage, config, and streaming API client.
> Layer 4 delivers the conversation runtime with `ToolExecutor` protocol.
> **v1.6 builds the concrete tool implementations that drive game mechanics.**
>
> Reference: the tool primitives (`ToolSpec`, `ToolRegistry`, `PermissionMode`,
> dispatch pipeline) are adapted from the [claw-code](https://github.com/ultraworkers/claw-code)
> Rust implementation, translated to Python for the RealTalk game context.

It catches exceptions and records them as error tool results. **Layer 5 provides the

## Architecture

### Why Tools?

The PRD specifies structured communication between the LLM and the game engine:

> **Response format:** `{ dialogue: string, mood_delta: { direction, intensity }, security_delta: { direction, intensity }, invite_turn: boolean }`

Rather than parsing free-form JSON from the LLM's text output, we use the Anthropic
**tool use** feature. The LLM calls game tools to deliver structured data. The
conversation runtime (Layer 4) already handles the tool loop — Layer 5 provides
what the tools actually *do*.

This design gives us:
- **Schema validation** at the API level (Anthropic rejects malformed tool inputs)
- **Reliable structured output** (no regex parsing of LLM text)
- **Clean separation** between LLM reasoning (text) and game state mutations (tools)
- **Extensibility** — new game mechanics are new tools, not new parsing logic

### Game State

Tools mutate a shared `GameState` object that tracks the live game:

```python
@dataclass
class GameState:
    """Mutable game state. Mutated by tool handlers, read by the game loop."""

    mood: int                      # 0–100, current value
    security: int                  # 0–100, current value
    turn_number: int               # incremented each game turn
    arc_active: bool               # True when Invitation Arc is in progress
    arc_turn: int | None           # turn number when arc was triggered
    game_result: str | None        # "win", "lose", or None (in progress)

    # Stage 1 delta (player reaction) — applied before response selection
    pending_mood_delta: int        # set by apply_reaction, consumed by game loop

    # Last deltas for display
    last_mood_direction: str       # "a" or "r"
    last_mood_intensity: int       # 1, 2, or 3
    last_security_direction: str   # "c" or "p"
    last_security_intensity: int   # 1, 2, or 3

    # Response options (set by generate_options, consumed by game loop)
    pending_options: list[str]     # exactly 3 strings

    @staticmethod
    def new(mood: int, security: int) -> GameState:
        """Create initial game state with starting mood and security values."""
        return GameState(
            mood=mood,
            security=security,
            turn_number=0,
            arc_active=False,
            arc_turn=None,
            game_result=None,
            pending_mood_delta=0,
            last_mood_direction="a",
            last_mood_intensity=0,
            last_security_direction="c",
            last_security_intensity=0,
            pending_options=[],
        )

    def clamp_mood(self) -> None:
        """Clamp mood to [0, 100]."""
        self.mood = max(0, min(100, self.mood))

    def clamp_security(self) -> None:
        """Clamp security to [0, 100]."""
        self.security = max(0, min(100, self.security))
```

The CLI layer creates `GameState.new(mood=random(30,50), security=random(40,60))`
at game start, matching the PRD ranges configured in `GameConfig`.

## `permissions.py` — Permission Model

Adapted from the Rust reference `PermissionMode` enum and `PermissionEnforcer`.
Simplified for the game context — no filesystem or bash permissions needed.

### What it owns

- `PermissionMode` — enum: `READ_ONLY`, `GAME_WRITE`, `CONTRIBUTOR_WRITE`
- `GamePermissionPolicy` — evaluates whether a tool call is permitted

### Implementation

```python
"""realtalk.permissions — Layer 5: permission model for game tools.

Permission is a scale: READ_ONLY < GAME_WRITE < CONTRIBUTOR_WRITE.
GamePermissionPolicy.check() is a pure function with no side effects.
The ToolRegistry calls it before every dispatch.

Dependencies: none (no imports from this project).
"""

from __future__ import annotations

from enum import Enum


class PermissionMode(Enum):
    """Permission levels, ordered by privilege.

    READ_ONLY:          tool reads game state but cannot mutate it.
    GAME_WRITE:         tool mutates GameState (mood, security, arc flags).
    CONTRIBUTOR_WRITE:  tool writes session data to disk (contributor mode).
    """

    READ_ONLY = "read-only"
    GAME_WRITE = "game-write"
    CONTRIBUTOR_WRITE = "contributor-write"

    def __le__(self, other: PermissionMode) -> bool:
        order = [
            PermissionMode.READ_ONLY,
            PermissionMode.GAME_WRITE,
            PermissionMode.CONTRIBUTOR_WRITE,
        ]
        return order.index(self) <= order.index(other)

    def __lt__(self, other: PermissionMode) -> bool:
        return self <= other and self != other


class PermissionDenied(Exception):
    """Raised when a tool call requires a higher permission level."""

    def __init__(self, tool_name: str, required: PermissionMode, active: PermissionMode) -> None:
        self.tool_name = tool_name
        self.required = required
        self.active = active
        super().__init__(
            f"tool '{tool_name}' requires {required.value}, "
            f"but active mode is {active.value}"
        )


class GamePermissionPolicy:
    """Evaluate whether a tool call is permitted under the active mode.

    Pure data object — no side effects. Pass to ToolRegistry; never
    prompt the user or fire hooks directly.

    The policy enforces: tool.required_permission <= active_mode.
    """

    def __init__(self, active_mode: PermissionMode) -> None:
        self._active = active_mode

    @property
    def active_mode(self) -> PermissionMode:
        return self._active

    def check(self, tool_name: str, required: PermissionMode) -> None:
        """Raise PermissionDenied if the tool's required permission exceeds active mode."""
        if not (required <= self._active):
            raise PermissionDenied(tool_name, required, self._active)
```

### Key design decisions

**Permission is a scale, not a boolean.** Matches the Rust reference where
`ReadOnly < WorkspaceWrite < DangerFullAccess`. For the game: `ReadOnly < GameWrite < ContributorWrite`.

**`GamePermissionPolicy` is a pure data object.** No side effects, no user prompts.
The Rust reference's `PermissionEnforcer.check()` returns `EnforcementResult::Allowed`
or `EnforcementResult::Denied` — our Python equivalent raises `PermissionDenied` or
returns `None`.

**Default active mode is `GAME_WRITE`.** Normal gameplay needs to mutate game state.
`CONTRIBUTOR_WRITE` is activated only when `--contributor` flag is passed.
`READ_ONLY` exists for spectator/replay modes (future).

## `hooks.py` — Hook Runner

Adapted from the Rust reference `HookRunner` and the v1.0 spec's hook protocol.
Shell-based hooks with exit-code protocol + built-in contributor data capture.

### What it owns

- `HookDecision` — enum: `ALLOW`, `ASK`, `DENY`
- `HookResult` — decision + reason string
- `HookRunner` — fires shell commands at `PreToolUse`, `PostToolUse`, `PostToolUseFailure`
- `ContributorCapture` — built-in Python hook for writing per-turn JSONL

### Implementation

```python
"""realtalk.hooks — Layer 5: pre/post tool hook runner.

Hooks are shell commands configured in HookConfig. Exit code protocol:
  0 = allow, 1 = ask, 2 = deny (pre-hooks only).
Post-hooks are fire-and-forget (subprocess.Popen, non-blocking).

Hook timeout: 10 seconds. Timeout → allow. A slow hook never blocks the game.

The ContributorCapture class provides a built-in Python hook for writing
per-turn JSONL when --contributor mode is active.

Dependencies: config.py (HookConfig, ContributorConfig).
"""

from __future__ import annotations

import json
import subprocess
import time
from dataclasses import dataclass
from enum import Enum
from pathlib import Path

from realtalk.config import ContributorConfig, HookConfig


class HookDecision(Enum):
    ALLOW = "allow"
    ASK = "ask"
    DENY = "deny"


@dataclass(frozen=True)
class HookResult:
    """Result of a pre-hook evaluation."""

    decision: HookDecision
    reason: str

    @staticmethod
    def allowed() -> HookResult:
        return HookResult(decision=HookDecision.ALLOW, reason="")

    @staticmethod
    def denied(reason: str) -> HookResult:
        return HookResult(decision=HookDecision.DENY, reason=reason)


_EXIT_TO_DECISION = {
    0: HookDecision.ALLOW,
    1: HookDecision.ASK,
    2: HookDecision.DENY,
}

HOOK_TIMEOUT_SECONDS = 10


class HookRunner:
    """Fire configured shell commands at tool lifecycle events.

    Pre-hooks run synchronously (blocking). Their exit code determines
    whether the tool call proceeds.

    Post-hooks run via Popen (non-blocking). The game does not wait.

    Failure hooks fire when a tool execution raises an exception.
    """

    def __init__(self, config: HookConfig) -> None:
        self._config = config

    def pre(self, tool_name: str, input_json: str) -> HookResult:
        """Run all PreToolUse hooks. First DENY wins. First ASK wins if no DENY."""
        if not self._config.pre_tool_use:
            return HookResult.allowed()

        ask_result: HookResult | None = None

        for cmd in self._config.pre_tool_use:
            decision, reason = self._run_sync(cmd, tool_name, input_json)
            if decision == HookDecision.DENY:
                return HookResult(decision=HookDecision.DENY, reason=reason)
            if decision == HookDecision.ASK and ask_result is None:
                ask_result = HookResult(decision=HookDecision.ASK, reason=reason)

        return ask_result or HookResult.allowed()

    def post(self, tool_name: str, input_json: str, output: str) -> None:
        """Fire all PostToolUse hooks (non-blocking)."""
        env = self._hook_env(tool_name, input_json, output=output)
        for cmd in self._config.post_tool_use:
            try:
                subprocess.Popen(  # noqa: S603
                    ["sh", "-c", cmd],
                    env=env,
                    stdout=subprocess.DEVNULL,
                    stderr=subprocess.DEVNULL,
                )
            except OSError:
                pass  # fire-and-forget: swallow launch failures

    def failure(self, tool_name: str, input_json: str, error: str) -> None:
        """Fire all PostToolUseFailure hooks (non-blocking)."""
        env = self._hook_env(tool_name, input_json, error=error)
        for cmd in self._config.post_tool_use_failure:
            try:
                subprocess.Popen(  # noqa: S603
                    ["sh", "-c", cmd],
                    env=env,
                    stdout=subprocess.DEVNULL,
                    stderr=subprocess.DEVNULL,
                )
            except OSError:
                pass

    def _run_sync(
        self, cmd: str, tool_name: str, input_json: str
    ) -> tuple[HookDecision, str]:
        """Run a single hook command synchronously. Returns (decision, reason)."""
        env = self._hook_env(tool_name, input_json)
        try:
            result = subprocess.run(  # noqa: S603
                ["sh", "-c", cmd],
                env=env,
                capture_output=True,
                text=True,
                timeout=HOOK_TIMEOUT_SECONDS,
            )
            decision = _EXIT_TO_DECISION.get(result.returncode, HookDecision.ALLOW)
            reason = result.stdout.strip()
            return decision, reason
        except subprocess.TimeoutExpired:
            return HookDecision.ALLOW, "hook timed out"
        except OSError:
            return HookDecision.ALLOW, "hook failed to run"

    @staticmethod
    def _hook_env(
        tool_name: str,
        input_json: str,
        output: str = "",
        error: str = "",
    ) -> dict[str, str]:
        """Build the environment dict passed to hook subprocesses."""
        import os

        env = dict(os.environ)
        env["REALTALK_TOOL_NAME"] = tool_name
        env["REALTALK_TOOL_INPUT"] = input_json
        env["REALTALK_TOOL_OUTPUT"] = output
        env["REALTALK_TOOL_ERROR"] = error
        return env


class ContributorCapture:
    """Built-in hook for writing per-turn data to JSONL in contributor mode.

    Called by ToolRegistry after every tool execution (same position as post-hook).
    Writes one JSON line per tool call. The JSONL file is at:
        {contributor.resolved_session_dir}/{session_id}.jsonl

    Privacy guarantee: player reaction ratings are NEVER included in the
    active LLM prompt. ContributorCapture only writes to local storage.
    """

    def __init__(self, config: ContributorConfig, session_id: str) -> None:
        self._enabled = config.enabled
        self._dir = config.resolved_session_dir
        self._session_id = session_id
        self._path: Path | None = None

    def _ensure_dir(self) -> Path:
        if self._path is None:
            self._dir.mkdir(parents=True, exist_ok=True)
            self._path = self._dir / f"{self._session_id}.jsonl"
        return self._path

    def capture(
        self,
        tool_name: str,
        input_json: str,
        output: str,
        is_error: bool,
        turn_number: int,
    ) -> None:
        """Append one JSONL line. No-op if contributor mode is disabled."""
        if not self._enabled:
            return

        path = self._ensure_dir()
        record = {
            "timestamp": time.time(),
            "turn": turn_number,
            "tool": tool_name,
            "input": input_json,
            "output": output,
            "is_error": is_error,
        }
        with open(path, "a") as f:
            f.write(json.dumps(record) + "\n")
```

### Key design decisions

**Exit code protocol matches v1.0 spec.** `0` = allow, `1` = ask, `2` = deny.
The Rust reference uses the same protocol. Unknown exit codes default to allow.

**Post-hooks are non-blocking.** `subprocess.Popen` — the turn never waits for
post-hooks. Same as the Rust reference. This is critical for game pacing.

**Hook timeout: 10 seconds.** Shorter than the Rust reference's 30 seconds because
the game is interactive. A slow hook should never stall the game loop. Timeout → allow.

**ContributorCapture is a Python object, not a shell hook.** Shell hooks are the
extension point; contributor capture is a first-party concern. It writes JSONL in the
same format as the PRD's contributor mode spec (§10.6).

**Environment variables for hook context.** Hooks receive `REALTALK_TOOL_NAME`,
`REALTALK_TOOL_INPUT`, `REALTALK_TOOL_OUTPUT`, `REALTALK_TOOL_ERROR`. Same pattern
as the Rust reference passing tool context via the process environment.

## `game_tools.py` — Game Tool Handlers

Five pure functions that implement game mechanics. Each takes parsed JSON input
and a `GameState` reference, performs the mutation, and returns a result string
that the LLM sees as the tool result.

### What it owns

- `GameState` — mutable game state dataclass (defined above)
- `handle_character_respond()` — apply mood + security deltas, return confirmation
- `handle_generate_options()` — validate and store 3 player response options
- `handle_apply_reaction()` — apply player's a/r rating (Stage 1 mood delta)
- `handle_trigger_invitation()` — activate the Invitation Arc
- `handle_evaluate_choice()` — score the player's arc response

### Implementation

```python
"""realtalk.game_tools — Layer 5: concrete game tool handler functions.

Each function takes:
  - input_data: dict (parsed from JSON)
  - game_state: GameState (mutable reference)
  - game_config: GameConfig (read-only, for thresholds and deltas)

Returns a string — the tool result the LLM sees.
Raises ValueError on invalid input (ToolRegistry catches and records as error).

Dependencies: config.py (GameConfig for reaction deltas, arc threshold).
"""

from __future__ import annotations

import json
from dataclasses import dataclass, field

from realtalk.config import GameConfig


@dataclass
class GameState:
    """Mutable game state. Mutated by tool handlers, read by the game loop."""

    mood: int
    security: int
    turn_number: int
    arc_active: bool
    arc_turn: int | None
    game_result: str | None

    pending_mood_delta: int
    last_mood_direction: str
    last_mood_intensity: int
    last_security_direction: str
    last_security_intensity: int

    pending_options: list[str] = field(default_factory=list)

    @staticmethod
    def new(mood: int, security: int) -> GameState:
        return GameState(
            mood=mood,
            security=security,
            turn_number=0,
            arc_active=False,
            arc_turn=None,
            game_result=None,
            pending_mood_delta=0,
            last_mood_direction="a",
            last_mood_intensity=0,
            last_security_direction="c",
            last_security_intensity=0,
            pending_options=[],
        )

    def clamp_mood(self) -> None:
        self.mood = max(0, min(100, self.mood))

    def clamp_security(self) -> None:
        self.security = max(0, min(100, self.security))


# ---------------------------------------------------------------------------
# Tool handlers
# ---------------------------------------------------------------------------

_VALID_MOOD_DIRECTIONS = {"a", "r"}
_VALID_SECURITY_DIRECTIONS = {"c", "p"}
_VALID_INTENSITIES = {1, 2, 3}


def handle_character_respond(
    input_data: dict[str, object],
    game_state: GameState,
    game_config: GameConfig,
) -> str:
    """LLM character delivers dialogue and updates mood + security bars.

    This is the Stage 2 update (after player reaction). The LLM determines
    how the character responds to the player's chosen response.

    Input fields:
        dialogue (str):            character's spoken line (1-4 sentences)
        mood_direction (str):      "a" (attraction) or "r" (repulsion)
        mood_intensity (int):      1, 2, or 3
        security_direction (str):  "c" (comfort) or "p" (protective)
        security_intensity (int):  1, 2, or 3
        narrative_cue (str):       body language / environmental detail (optional)
        invite_turn (bool):        True if this turn is an Invitation Arc turn

    Returns: JSON string with updated mood, security, and arc status.
    """
    dialogue = str(input_data["dialogue"])
    mood_dir = str(input_data["mood_direction"])
    mood_int = int(input_data["mood_intensity"])  # type: ignore[arg-type]
    sec_dir = str(input_data["security_direction"])
    sec_int = int(input_data["security_intensity"])  # type: ignore[arg-type]
    invite = bool(input_data["invite_turn"])

    if mood_dir not in _VALID_MOOD_DIRECTIONS:
        raise ValueError(f"mood_direction must be 'a' or 'r', got '{mood_dir}'")
    if mood_int not in _VALID_INTENSITIES:
        raise ValueError(f"mood_intensity must be 1, 2, or 3, got {mood_int}")
    if sec_dir not in _VALID_SECURITY_DIRECTIONS:
        raise ValueError(f"security_direction must be 'c' or 'p', got '{sec_dir}'")
    if sec_int not in _VALID_INTENSITIES:
        raise ValueError(f"security_intensity must be 1, 2, or 3, got {sec_int}")

    # Apply mood delta (Stage 2)
    mood_delta = game_config.reaction_delta(mood_int)
    if mood_dir == "r":
        mood_delta = -mood_delta
    game_state.mood += mood_delta
    game_state.clamp_mood()

    # Apply security delta
    sec_delta = game_config.reaction_delta(sec_int)
    if sec_dir == "p":
        sec_delta = -sec_delta
    game_state.security += sec_delta
    game_state.clamp_security()

    # Track last deltas for display
    game_state.last_mood_direction = mood_dir
    game_state.last_mood_intensity = mood_int
    game_state.last_security_direction = sec_dir
    game_state.last_security_intensity = sec_int

    # Handle Invitation Arc trigger
    if invite and game_state.mood >= game_config.arc_trigger_threshold:
        game_state.arc_active = True
        if game_state.arc_turn is None:
            game_state.arc_turn = game_state.turn_number

    # If mood drops below threshold during active arc, suspend it
    if game_state.arc_active and game_state.mood < game_config.arc_trigger_threshold:
        game_state.arc_active = False

    # Check for mood-zero lose condition
    if game_state.mood <= 0:
        game_state.game_result = "lose"

    return json.dumps({
        "mood": game_state.mood,
        "security": game_state.security,
        "arc_active": game_state.arc_active,
        "game_result": game_state.game_result,
        "dialogue": dialogue,
    })


def handle_generate_options(
    input_data: dict[str, object],
    game_state: GameState,
    game_config: GameConfig,
) -> str:
    """LLM produces 3 differentiated player response options.

    Options must be:
    - Exactly 3 strings
    - Each under 25 words
    - Written in first person
    - Meaningfully differentiated (open / deflect / withdraw)

    Input fields:
        options (list[str]): exactly 3 response option strings

    Returns: JSON confirmation with option count.
    """
    options = input_data.get("options")
    if not isinstance(options, list) or len(options) != 3:
        raise ValueError(f"options must be a list of exactly 3 strings, got {type(options)}")

    validated: list[str] = []
    for i, opt in enumerate(options):
        text = str(opt)
        word_count = len(text.split())
        if word_count > 25:
            raise ValueError(
                f"option {i + 1} has {word_count} words (max 25): '{text[:50]}...'"
            )
        validated.append(text)

    game_state.pending_options = validated

    return json.dumps({
        "options_stored": len(validated),
        "options": validated,
    })


def handle_apply_reaction(
    input_data: dict[str, object],
    game_state: GameState,
    game_config: GameConfig,
) -> str:
    """Apply the player's a/r + intensity rating to the mood bar (Stage 1).

    This is called BEFORE response selection. The player rates their internal
    reaction, and it immediately affects the mood bar. The raw rating is
    NEVER sent to the LLM — this tool is called by the game loop, not by
    the model.

    Input fields:
        direction (str): "a" (attraction) or "r" (repulsion)
        intensity (int): 1 (low), 2 (medium), or 3 (high)

    Returns: JSON with new mood value and the delta applied.
    """
    direction = str(input_data["direction"])
    intensity = int(input_data["intensity"])  # type: ignore[arg-type]

    if direction not in _VALID_MOOD_DIRECTIONS:
        raise ValueError(f"direction must be 'a' or 'r', got '{direction}'")
    if intensity not in _VALID_INTENSITIES:
        raise ValueError(f"intensity must be 1, 2, or 3, got {intensity}")

    delta = game_config.reaction_delta(intensity)
    if direction == "r":
        delta = -delta

    game_state.mood += delta
    game_state.clamp_mood()
    game_state.pending_mood_delta = delta

    return json.dumps({
        "mood": game_state.mood,
        "delta": delta,
        "direction": direction,
        "intensity": intensity,
    })


def handle_trigger_invitation(
    input_data: dict[str, object],
    game_state: GameState,
    game_config: GameConfig,
) -> str:
    """Signal entry into the Invitation Arc.

    The LLM calls this when mood >= arc_trigger_threshold and the character's
    dialogue has been building toward a genuine opening over 2-4 turns.

    The arc can be suspended if mood drops below threshold. It resumes when
    mood crosses the threshold again.

    Input fields:
        arc_turn_offset (int): how many turns until the Invitation Turn (1-4)

    Returns: JSON with arc status.
    """
    offset = int(input_data.get("arc_turn_offset", 2))  # type: ignore[arg-type]
    if not (1 <= offset <= 4):
        raise ValueError(f"arc_turn_offset must be 1-4, got {offset}")

    if game_state.mood < game_config.arc_trigger_threshold:
        return json.dumps({
            "arc_active": False,
            "reason": f"mood {game_state.mood} < threshold {game_config.arc_trigger_threshold}",
        })

    game_state.arc_active = True
    game_state.arc_turn = game_state.turn_number + offset

    return json.dumps({
        "arc_active": True,
        "arc_turn": game_state.arc_turn,
        "current_turn": game_state.turn_number,
    })


def handle_evaluate_choice(
    input_data: dict[str, object],
    game_state: GameState,
    game_config: GameConfig,
) -> str:
    """Score the player's response during the Invitation Arc.

    Called on the designated Invitation Turn. The LLM evaluates whether
    the player's chosen response meets the character's emotional offer
    with presence, not performance.

    Input fields:
        choice_quality (str): "attuned", "deflecting", or "overclaiming"

    Returns: JSON with game result ("win", "lose", or "continue").
    """
    quality = str(input_data["choice_quality"])
    valid_qualities = {"attuned", "deflecting", "overclaiming"}

    if quality not in valid_qualities:
        raise ValueError(
            f"choice_quality must be one of {valid_qualities}, got '{quality}'"
        )

    if quality == "attuned":
        game_state.game_result = "win"
    else:
        game_state.game_result = "lose"

    return json.dumps({
        "game_result": game_state.game_result,
        "choice_quality": quality,
        "final_mood": game_state.mood,
        "final_security": game_state.security,
    })
```

### Key design decisions

**`GameState` is a plain mutable dataclass.** Unlike `Session` (frozen, event-sourced),
`GameState` is designed for rapid in-place mutation during the turn loop. The session
owns the event log; `GameState` owns the live values. This separation mirrors the Rust
reference where the tool handler mutates state directly and the session records the events.

**Tool handlers are pure functions, not methods.** They take `(input_data, game_state, game_config)`
and return a string. This makes them trivially testable — no class instantiation needed. The
`ToolRegistry` owns the wiring.

**`apply_reaction` is not LLM-callable.** The PRD says: "The raw rating is **never sent
to the LLM.**" This tool is called by the game loop directly (before response selection),
not by the model. The `ToolRegistry` dispatches it like any other tool, but it's excluded
from `GAME_TOOL_SPECS` (the schemas sent to the LLM).

**`character_respond` does double duty.** It applies the Stage 2 mood/security deltas AND
returns the dialogue. This matches the PRD's response format: a single structured object with
dialogue + deltas + invite flag.

**Validation is strict.** Invalid directions, intensities, or option counts raise `ValueError`.
The conversation runtime catches these and records them as error tool results. The LLM sees
the error and can retry.

## `tools.py` — Tool Registry & Specs

The dispatcher. Wires `GamePermissionPolicy`, `HookRunner`, and tool handlers into a
single `execute(tool_name, input_json) -> str` that implements `ToolExecutor` from Layer 4.

### What it owns

- `ToolSpec` — frozen dataclass: name, description, input_schema, required_permission
- `GAME_TOOL_SPECS` — the 4 LLM-callable tool definitions (schemas passed to the model)
- `ALL_TOOL_SPECS` — all 5 tool specs (including `apply_reaction` for internal use)
- `ToolRegistry` — the dispatcher class implementing `ToolExecutor`

```python
class ToolRegistry:
    """Wires permissions, hooks, and tool handlers into a single dispatch point.

    Implements the ToolExecutor protocol from conversation.py:
        def execute(self, tool_name: str, tool_input: str) -> str

    Mirrors the Rust reference GlobalToolRegistry:
        pub fn execute(&self, name: &str, input: &Value) -> Result<String, String>

    Construction:
        registry = ToolRegistry(
            game_state=GameState.new(mood=42, security=55),
            config=config,
            session_id="abc123",
        )
        runtime = ConversationRuntime(
            ...,
            tool_executor=registry,
            tool_definitions=registry.tool_definitions(),
            ...
        )
    """

    def __init__(
        self,
        game_state: GameState,
        config: RuntimeConfig,
        session_id: str = "",
    ) -> None:
        self._game_state = game_state
        self._game_config = config.game
        self._policy = GamePermissionPolicy(
            active_mode=PermissionMode.CONTRIBUTOR_WRITE
            if config.contributor.enabled
            else PermissionMode.GAME_WRITE
        )
        self._hooks = HookRunner(config.hooks)
        self._contributor = ContributorCapture(config.contributor, session_id)
        self._spec_map = {spec.name: spec for spec in ALL_TOOL_SPECS}

    @property
    def game_state(self) -> GameState:
        """Current game state (mutable reference)."""
        return self._game_state

    def execute(self, tool_name: str, tool_input: str) -> str:
        """Dispatch a tool call through the full pipeline.

        Pipeline:
          1. Lookup spec (unknown → error string, not exception)
          2. Permission check (denied → PermissionDenied exception)
          3. Pre-hook (deny → error string)
          4. Parse input JSON
          5. Execute handler
          6. Post-hook (fire-and-forget)
          7. Contributor capture (if enabled)
          8. Return result string
        """
        # 1. Lookup
        spec = self._spec_map.get(tool_name)
        if spec is None:
            return f"ERROR: unknown tool {tool_name}"

        # 2. Permission check (raises PermissionDenied on failure)
        self._policy.check(tool_name, spec.required_permission)

        # 3. Pre-hook
        hook_result = self._hooks.pre(tool_name, tool_input)
        if hook_result.decision == HookDecision.DENY:
            return f"ERROR: blocked by hook: {hook_result.reason}"

        # 4. Parse input
        input_data: dict[str, object] = json.loads(tool_input) if tool_input else {}

        # 5. Execute handler
        handler = _HANDLER_MAP[tool_name]
        result = handler(input_data, self._game_state, self._game_config)

        # 6. Post-hook (fire-and-forget)
        self._hooks.post(tool_name, tool_input, result)

        # 7. Contributor capture
        self._contributor.capture(
            tool_name=tool_name,
            input_json=tool_input,
            output=result,
            is_error=False,
            turn_number=self._game_state.turn_number,
        )

        return result

    def tool_definitions(self) -> list[dict[str, object]]:
        """Return Anthropic-format tool definitions for ApiRequest.tools.

        Only returns LLM-callable tools (GAME_TOOL_SPECS). Internal tools
        like apply_reaction are excluded.
        """
        return [
            {
                "name": spec.name,
                "description": spec.description,
                "input_schema": spec.input_schema,
            }
            for spec in GAME_TOOL_SPECS
        ]

    def execute_reaction(self, direction: str, intensity: int) -> str:
        """Convenience method for the game loop to apply player reaction.

        Bypasses the LLM — called directly by the CLI game loop after the
        player inputs their a/r + intensity rating.
        """
        input_json = json.dumps({"direction": direction, "intensity": intensity})
        return self.execute("apply_reaction", input_json)
```

### Key design decisions

**`GAME_TOOL_SPECS` vs `ALL_TOOL_SPECS`.** Only `GAME_TOOL_SPECS` (4 tools) are passed
to the LLM via `tool_definitions()`. `apply_reaction` is internal-only — the game loop
calls it directly via `execute_reaction()`. This enforces the PRD privacy guarantee:
the player's reaction rating is never in the LLM prompt.

**Dispatch is a plain dict.** `{name: handler}` — same as the v1.0 spec: "No dynamic
registration, no plugin system yet." Matches the Rust reference `match name { ... }`.

**Pipeline order is fixed.** Permission check → pre-hook → execute → post-hook →
contributor capture. Same as the Rust reference. Never skip a stage.

**Unknown tool returns error string, not exception.** The Rust reference returns
`Err(format!("unsupported tool: {name}"))` which the model receives as a tool result.
Our Python equivalent returns `"ERROR: unknown tool {name}"`. The model can react.

**`ToolRegistry` implements `ToolExecutor` structurally.** It has an `execute(tool_name, tool_input) -> str`
method. No need to inherit or register — Python's `@runtime_checkable` Protocol from Layer 4
matches structurally.

## Integration with Layer 4

The CLI layer constructs everything and wires it together:

```python
# In cli.py (sketch — full implementation is Layer 8)

import random
from realtalk.config import ConfigLoader
from realtalk.game_tools import GameState
from realtalk.tools import ToolRegistry
from realtalk.conversation import ConversationRuntime
from realtalk.api import AnthropicClient

def _build_game_runtime(session, config):
    """Construct the runtime with game tools wired in."""

    game_state = GameState.new(
        mood=random.randint(config.game.mood_start_min, config.game.mood_start_max),
        security=random.randint(config.game.security_start_min, config.game.security_start_max),
    )

    registry = ToolRegistry(
        game_state=game_state,
        config=config,
        session_id=session.session_id,
    )

    client = AnthropicClient(
        model=config.game.model,
        max_tokens=config.game.max_tokens,
        api_key=config.api_key,
    )

    runtime = ConversationRuntime(
        api_client=client,
        tool_executor=registry,           # ToolRegistry implements ToolExecutor
        session=session,
        system_prompt=build_system_prompt(...),
        tool_definitions=registry.tool_definitions(),  # 4 LLM-callable tools
        on_text=print,
        model=config.game.model,
        max_tokens=config.game.max_tokens,
        temperature=config.game.temperature,
    )

    return runtime, registry
```

### Game Turn Flow (with tools)

```
Player sees character's message
  ↓
Player rates reaction: "a2"
  ↓
Game loop calls: registry.execute_reaction("a", 2)
  → apply_reaction tool → mood bar updates (Stage 1)
  ↓
Player sees 3 options (from pending_options)
Player chooses option 2
  ↓
Game loop calls: runtime.run_turn(chosen_option_text)
  ↓
ConversationRuntime streams LLM response
  → LLM calls character_respond tool (Stage 2 mood/security update)
  → LLM calls generate_options tool (next turn's options)
  → LLM may call trigger_invitation (if mood ≥ 80)
  → LLM may call evaluate_choice (if arc_turn reached)
  ↓
Game loop reads: registry.game_state
  → renders mood/security bars
  → checks game_result for win/lose
  → displays pending_options for next turn
```

## Non-Goals for v1.6

- **Bash tool / file ops** — RealTalk is a game, not a coding agent. No filesystem
  tools needed at runtime.
- **MCP integration** — external tool servers are deferred to v2.0.
- **Dynamic tool registration** — all tools are statically defined. Plugin system is v2.0.
- **Async tool execution** — game loop is single-threaded. Async tools are v2.0.
- **Tool search / deferred loading** — all 5 tools are loaded at startup. The Rust
  reference's `ToolSearch` and deferred tool mechanism is unnecessary for 5 tools.

## Questions Resolved

| # | Question | Resolution |
|---|----------|-----------|
| Q1 | Should the LLM call tools or return structured JSON in text? | Tools. Schema validation, reliable parsing, clean separation. |
| Q2 | Is `apply_reaction` an LLM tool or a game-loop function? | Both — it's in the registry for uniform dispatch, but excluded from `tool_definitions()` sent to the LLM. PRD: "The raw rating is **never sent to the LLM.**" |
| Q3 | Should `GameState` be frozen like `Session`? | No. Session is an event log (append-only); GameState is live values (mutated in-place). Different concerns, different data models. |
| Q4 | How many tools does the LLM see? | 4: `character_respond`, `generate_options`, `trigger_invitation`, `evaluate_choice`. |
| Q5 | What happens when the LLM calls an invalid tool? | Error string returned: `"ERROR: unknown tool {name}"`. No exception. The LLM can react. |
| Q6 | Should hooks be shell commands or Python callables? | Shell commands (v1.0 spec pattern), plus one built-in Python `ContributorCapture`. |
