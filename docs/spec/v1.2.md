# Eng Review: realtalk v1.2 — Persistence + Config

## Context

Reviewing `docs/spec/v1.2.md` before implementation. The spec adds two layers:
- Layer 1: `storage.py` — physical session persistence (JSONL append, rotation, crash safety)
- Layer 2: `config.py` — typed immutable config using chz + pydantic

Design doc approved at `docs/design/v1.2-design.md` with Approach B (spec + training-aware archive).

User's specific concerns: persistence trade-offs, LLM training dataset compatibility, config design decisions.

## Step 0: Scope Challenge — Scope accepted as-is

**Existing code that partially solves sub-problems:**
- `session.py:1257` — `session_to_jsonl()` / `session_from_jsonl()` already handles serialization
- `session.py:1084` — `event_to_dict()` / `event_from_dict()` for individual events
- `session.py:1363-1552` — Training exports (SFT, preference, trajectory) already exist

**Minimum change set:** 4 new files (storage.py, config.py, test_storage.py, test_config.py), pyproject.toml update.

**Complexity check:** 4 files, ~6 new classes. At the boundary but justified — storage and config are genuinely separate concerns.

## Review Findings

### Architecture — 4 issues found, all resolved

1. **[P1] (confidence: 9/10) JSONL error handling seam** — `session_from_jsonl` (`session.py:1296`) raises `SerializationError` on invalid JSON, but `StoredSession.load()` needs to skip corrupt lines for crash safety. **Resolution:** Add `skip_errors=False` parameter to `session_from_jsonl`. Storage calls with `skip_errors=True`.

2. **[P1] (confidence: 9/10) SessionStarted missing in rotated files** — After rotation, files other than the oldest won't have `SessionStarted`, breaking `session_from_jsonl` and making archived files useless for training. **Resolution:** Copy `SessionStarted` header event as the first line of every new file during rotation. Every file is self-describing. Critical for training data compatibility.

3. **[P2] (confidence: 8/10) Opaque workspace fingerprints** — `sha256` directory names are unreadable. Bad for debugging and training export. **Resolution:** Use `{dirname}-{hash[:8]}` format (e.g., `game-session-a3f2c8b1/`).

4. **[P2] (confidence: 7/10) Duplicate session path defaults** — `ContributorConfig.session_dir` and `SessionStore.root` both default to `~/.realtalk/sessions` with no shared constant. **Resolution:** Extract `DEFAULT_SESSION_ROOT` constant referenced by both.

### Code Quality — 3 issues found, all resolved

5. **[P1] (confidence: 9/10) DRY violation: duplicated defaults** — Every config field defined twice (Raw pydantic + chz class). **Resolution:** Single source of truth in pydantic models. Remove hardcoded defaults from chz classes. Tests use `chz.replace()` on a default-loaded config.

6. **[P2] (confidence: 8/10) Dead PermissionMode enum** — Defined but unused. **Resolution:** Remove from v1.2. Add in v0.3+ when tool-use permissions are wired.

7. **[P2] (confidence: 7/10) GameConfig.reaction_delta KeyError** — Dict lookup raises unhelpful `KeyError` on invalid intensity. **Resolution:** Add explicit `if intensity not in (1, 2, 3): raise ValueError(...)`.

## NOT in scope

- **Server/multi-user/cloud sync** — v1.2 is local-first, single-player.
- **Archive growth policy** — Deferred to v0.3. Archive grows unbounded for now.
- **PermissionMode and tool-use permissions** — Removed from v1.2.
- **CLI wiring (chz.nested_entrypoint)** — Spec previews it for v0.3, not built here.
- **Incremental/lazy loading** — load() performance acceptable for v1.2 session sizes.
- **Training export pipeline tooling** — session.py exports exist. No CLI or batch tooling in v1.2.

## Failure Modes

| Codepath | Failure scenario | Test? | Error handling? | User sees? |
|---|---|---|---|---|
| `append()` | Crash mid-write → truncated last line | YES (spec) | YES (skip on load) | Silent recovery |
| `rotate_if_needed()` | Crash during rename chain → .1 and session.jsonl both exist | NO | NO | **CRITICAL GAP** — data duplication on reload |
| `load()` across rotated files | Missing .2 file (disk error) → gap in event stream | NO | NO | Silent data loss |
| `ConfigLoader._load_raw()` | Malformed JSON in .realtalk.json | NO | NO | Unhandled exception |
| `RuntimeConfig.api_key` | Empty string ANTHROPIC_API_KEY | NO | NO | Empty key passed to Anthropic client |
| `archive_before_rotate()` | Disk full → can't copy to archive | NO | NO | Silent archive failure |

**Critical gap:** Crash during rotation rename chain (rotate_if_needed). If the process dies between renaming `session.jsonl` → `session.jsonl.1` and creating the new `session.jsonl`, the system could lose data or have inconsistent state. Need either: (a) atomic rotation (rename to temp first), or (b) a recovery check on startup that detects and fixes partial rotation state.

### Key implementation details

**storage.py architecture:**
```
SessionStore
├── __init__(root: Path = DEFAULT_SESSION_ROOT)
├── session_dir(cwd) → Path        # {dirname}-{hash[:8]}/
├── session_path(cwd) → Path
├── list_sessions() → list[Path]
└── list_archived_sessions() → list[Path]

StoredSession
├── __init__(path: Path, max_bytes: int = 256*1024)
├── append(event: AnyEvent)         # writes event_to_dict as JSONL line
├── load() → Session                # calls session_from_jsonl(skip_errors=True)
└── exists() → bool

rotate_if_needed(path, max_bytes, keep)
  1. Check file size
  2. If >= max_bytes:
     a. Archive .{keep} if it exists (copy to archive dir)
     b. Read SessionStarted from first line of oldest file
     c. Cascade rename: .{keep-1}→.{keep}, ... .1→.2, path→.1
     d. Write SessionStarted header to new empty path
```

**config.py architecture:**
```
RawGameConfig (pydantic) → GameConfig (chz, no defaults)
RawContributorConfig → ContributorConfig
RawDisplayConfig → DisplayConfig  
RawHookConfig → HookConfig
RawRuntimeConfig → RuntimeConfig

ConfigLoader
├── load() → RuntimeConfig
├── _load_raw() → dict (three-tier file merge)
└── _to_chz(raw) → RuntimeConfig

_deep_merge(base, override)  # lists replace, dicts merge
```
