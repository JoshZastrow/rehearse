## Design Decisions

### DRY: single source of truth for defaults

Every config field should be defined once. Defaults live in the pydantic model;
the chz class receives all values via `_to_chz()` and carries no defaults of its own.

```python
# BEFORE (bad — duplicated defaults):
class RawGameConfig(BaseModel):
    model: str = "claude-haiku-4-5-20251001"

@chz.chz
class GameConfig:
    model: str = "claude-haiku-4-5-20251001"  # duplicated!

# AFTER (good — single source of truth):
class RawGameConfig(BaseModel):
    model: str = "claude-haiku-4-5-20251001"  # defaults live here only

@chz.chz
class GameConfig:
    model: str  # no default — populated by _to_chz() from validated pydantic
```

Tests that need a config object use `ConfigLoader(cwd=tmp_path).load()` or
`chz.replace()` on a loaded config, never direct `GameConfig()` construction.

### PermissionMode enum is out of scope

`PermissionMode` is not referenced by any config field in v1.2. It belongs in v1.3+
when tool-use permissions are wired.

### GameConfig.reaction_delta validation

The dict lookup `{1: ..., 2: ..., 3: ...}[intensity]` raises an unhelpful `KeyError`
on invalid intensity values. Use explicit validation instead:

```python
def reaction_delta(self, intensity: int) -> int:
    if intensity not in (1, 2, 3):
        raise ValueError(f"intensity must be 1, 2, or 3, got {intensity}")
    return {1: self.reaction_delta_low,
            2: self.reaction_delta_medium,
            3: self.reaction_delta_high}[intensity]
```

### Cross-field validation on pydantic models

Add `model_validator` to `RawGameConfig` to catch invalid range combinations at parse time:

```python
from pydantic import model_validator

class RawGameConfig(BaseModel):
    ...

    @model_validator(mode="after")
    def check_ranges(self) -> RawGameConfig:
        if self.mood_start_min > self.mood_start_max:
            raise ValueError(
                f"mood_start_min ({self.mood_start_min}) must be <= "
                f"mood_start_max ({self.mood_start_max})"
            )
        if self.security_start_min > self.security_start_max:
            raise ValueError(
                f"security_start_min ({self.security_start_min}) must be <= "
                f"security_start_max ({self.security_start_max})"
            )
        if self.turn_hard_cap < 1:
            raise ValueError(f"turn_hard_cap must be >= 1, got {self.turn_hard_cap}")
        if self.min_turns_to_win < 1:
            raise ValueError(f"min_turns_to_win must be >= 1, got {self.min_turns_to_win}")
        return self
```

### Deep merge: lists replace, dicts merge

When two config tiers set the same list key, the higher-priority tier's list replaces
the lower tier's entirely. Dicts are merged recursively. This matches git/npm config behavior.

### ConfigLoader._load_raw handles bad JSON gracefully

Malformed JSON and non-dict JSON in config files are silently skipped, not a crash.
The loader wraps each file read in `try/except (json.JSONDecodeError, OSError)` and
checks `isinstance(data, dict)` before merging.

### api_key is a lazy property

`@chz.init_property` evaluates eagerly at construction time. Using it for `api_key`
causes `ConfigLoader.load()` to raise when `ANTHROPIC_API_KEY` is absent, which is
wrong — the key should only be required when the game actually makes an API call.
Use a regular `@property` so `load()` always succeeds:

```python
@chz.chz
class RuntimeConfig:
    ...

    @property
    def api_key(self) -> str:
        key = os.environ.get("ANTHROPIC_API_KEY", "")
        if not key:
            raise EnvironmentError("ANTHROPIC_API_KEY is not set.")
        return key
```

An empty string is treated as missing (`""` is falsy in Python, so `if not key` catches it).

## Public API

```python
from pathlib import Path
import chz
from pydantic import BaseModel, field_validator, model_validator


# --- Pydantic input models (validation boundary) ---

class RawGameConfig(BaseModel):
    model: str = "claude-haiku-4-5-20251001"
    min_turns_to_win: int = 8
    turn_hard_cap: int = 25
    arc_trigger_threshold: int = 80
    mood_start_min: int = 30
    mood_start_max: int = 50
    security_start_min: int = 40
    security_start_max: int = 60
    reaction_delta_low: int = 3
    reaction_delta_medium: int = 7
    reaction_delta_high: int = 12

    @field_validator("arc_trigger_threshold")
    @classmethod
    def arc_threshold_in_range(cls, v: int) -> int: ...

    @model_validator(mode="after")
    def check_ranges(self) -> RawGameConfig: ...

class RawContributorConfig(BaseModel): ...
class RawDisplayConfig(BaseModel): ...
class RawHookConfig(BaseModel): ...
class RawRuntimeConfig(BaseModel): ...


# --- chz config objects (no scalar defaults — populated by _to_chz) ---

@chz.chz
class GameConfig:
    model: str
    min_turns_to_win: int
    turn_hard_cap: int
    arc_trigger_threshold: int
    mood_start_min: int
    mood_start_max: int
    security_start_min: int
    security_start_max: int
    reaction_delta_low: int
    reaction_delta_medium: int
    reaction_delta_high: int

    def reaction_delta(self, intensity: int) -> int: ...

@chz.chz
class ContributorConfig:
    enabled: bool
    session_dir: str

    @chz.init_property
    def resolved_session_dir(self) -> Path: ...

@chz.chz
class DisplayConfig:
    no_color: bool
    debug: bool

@chz.chz
class HookConfig:
    pre_tool_use: list[str]      # chz.field(default_factory=list)
    post_tool_use: list[str]
    post_tool_use_failure: list[str]

@chz.chz
class RuntimeConfig:
    game: GameConfig
    contributor: ContributorConfig
    display: DisplayConfig
    hooks: HookConfig

    @property
    def api_key(self) -> str: ...   # lazy — raises EnvironmentError when key missing


# --- Loader ---

class ConfigLoader:
    def __init__(self, cwd: Path = Path.cwd()) -> None: ...
    def load(self) -> RuntimeConfig: ...
    def _load_raw(self) -> dict: ...
    def _to_chz(self, raw: RawRuntimeConfig) -> RuntimeConfig: ...

def _deep_merge(base: dict, override: dict) -> dict: ...
```

```

```

def test_defaults_load_cleanly(tmp_path, monkeypatch):
    """Default config loads with expected values."""
    monkeypatch.setenv("ANTHROPIC_API_KEY", "sk-test")
    config = ConfigLoader(cwd=tmp_path).load()
    assert config.game.arc_trigger_threshold == 80
    assert config.game.min_turns_to_win == 8
    assert config.contributor.enabled is False
    assert config.display.no_color is False


def test_three_tier_merge(tmp_path, monkeypatch):
    """Project tier overrides defaults, unset fields keep defaults."""
    monkeypatch.setenv("ANTHROPIC_API_KEY", "sk-test")
    (tmp_path / ".realtalk.json").write_text('{"game": {"arc_trigger_threshold": 75}}')
    config = ConfigLoader(cwd=tmp_path).load()
    assert config.game.arc_trigger_threshold == 75
    assert config.game.min_turns_to_win == 8


def test_local_tier_wins(tmp_path, monkeypatch):
    """Local tier overrides project tier."""
    monkeypatch.setenv("ANTHROPIC_API_KEY", "sk-test")
    (tmp_path / ".realtalk.json").write_text('{"game": {"model": "claude-sonnet-4-6"}}')

    (tmp_path / ".realtalk" / "settings.local.json").write_text(

    assert config.game.model == "claude-opus-4-6"

## Implementation Notes

1. **Remove `PermissionMode` enum.** It's unused in v1.2.

2. **Remove defaults from chz classes.** Defaults live only in pydantic models. chz fields should be bare type annotations (except collection types needing `chz.field(default_factory=...)`).

3. **Add `model_validator` to `RawGameConfig`.** Cross-field validation for mood/security ranges and positive turn caps.

4. **Add `ValueError` to `reaction_delta`.** Replace the silent `KeyError`.

5. **Use `@property` for `api_key`, not `@chz.init_property`.** `init_property` is eager.

6. **Verify `_load_raw` gracefully handles bad JSON.** The try/except already covers this. Add tests to confirm.

7. **No imports from session.py or storage.py.** Config is fully independent.

8. **No filesystem mocking.** All tests use real disk I/O via `tmp_path` and `monkeypatch`.
