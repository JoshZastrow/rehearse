# Realtalk — Python Build Spec v0.2

> Builds on top of [v0.1](./v0.1.md).
> v0.1 established the event-sourced session data types (`session.py`).
> v0.2 closes out the persistence layer and adds the configuration layer.
> The configuration layer is built using [`chz`](https://github.com/openai/chz) —
> OpenAI's declarative, immutable configuration library — on top of pydantic data models.

## What Was Built in v0.1 (Layer 0)

`realtalk/session.py` is complete. It owns the full event-sourced data model:

- Prefixed ID generators (`sess_`, `evt_`, `turn_`, `msg_`, `fb_`, ...)
- Enums: `MessageRole`, `TurnStatus`, `FeedbackSource`
- Content parts: `TextPart`, `ToolCallPart`, `ToolResultPart`
- Seven canonical event types: `SessionStarted`, `TurnStarted`, `MessageAdded`,
  `ToolCallRecorded`, `ToolResultRecorded`, `FeedbackRecorded`, `TurnEnded`,
  `SessionMetadataUpdated`
- Immutable `Session` dataclass (frozen, append returns a new instance)
- Derived projections: `derive_messages`, `derive_turns`, `derive_feedback`
- JSONL serialization: `event_to_dict` / `event_from_dict`, `session_to_jsonl` / `session_from_jsonl`
- Training exports: `to_sft_examples`, `to_preference_examples`, `to_trajectory_examples`
- Full validation: `validate_session` (20 structural + behavioral invariants)
- 71 tests passing (unit + doctests + Hypothesis property tests)

**What session.py does NOT own:** physical disk I/O, file path management, log rotation,
or workspace isolation. Those belong to `storage.py`.

## Layer 1 — `storage.py`: Physical Session Persistence

**Depends on:** `session.py` (Layer 0) only.

### What it owns

- `SessionStore` — workspace-fingerprinted directory manager
- `StoredSession` — live handle to a session file on disk; wraps a `Session` and its path
- `append_event` — crash-safe append of a single event to the JSONL file
- `load_session` — reconstruct a `Session` by reading and replaying a JSONL file
- `rotate_if_needed` — enforce the 256KB / 3-file rotation policy
- `list_sessions` — enumerate stored sessions under a `SessionStore` root

### Design decisions

**Workspace fingerprint for isolation.**  
`sha256(Path(cwd).resolve().as_posix().encode())[:16]` becomes the subdirectory name
under `~/.realtalk/sessions/`. Two games in different working directories never share a
session file. Auto-resume is directory-scoped — no session ID argument required.

```
~/.realtalk/sessions/
  a3f2c8b1e0d94f72/    ← sha256 of /Users/alice/projects/game-session
    session.jsonl
    session.jsonl.1    ← rotated
    session.jsonl.2
```

**Append-only writes are crash-safe.**  
Each call to `append_event` opens the file in append mode, writes one JSON line + `\n`,
and flushes. A crash mid-write corrupts at most the final incomplete line, which is
detectable during `load_session` (skip lines that do not parse as valid JSON).

**`load_session` reads rotated files in order.**  
When loading, read `session.jsonl.N` (oldest) through `session.jsonl` (newest), replay all
events in sequence. This makes log rotation invisible to the caller.

**Rotation policy:** 256KB active file, max 3 rotated files.  
Before each append, check file size. If ≥ 256KB:
1. Delete `session.jsonl.3` if it exists
2. Rename `.2` → `.3`, `.1` → `.2`, `session.jsonl` → `session.jsonl.1`
3. Create a new empty `session.jsonl`

**`StoredSession` is the live game handle.**  
The game loop holds a `StoredSession` and calls `stored.append(event)` each turn.
`StoredSession` owns no in-memory message list — it is purely a disk-append wrapper.
The in-memory `Session` is separate and updated by the caller.

### Public API

```python
from pathlib import Path
from realtalk.session import Session, AnyEvent

class SessionStore:
    """Manages session file paths under a root directory, keyed by workspace fingerprint."""

    def __init__(self, root: Path = Path.home() / ".realtalk" / "sessions") -> None: ...

    def session_dir(self, cwd: Path) -> Path:
        """Return (and create if needed) the directory for the given working directory."""
        ...

    def session_path(self, cwd: Path) -> Path:
        """Return the path to the active session.jsonl for the given working directory."""
        ...

    def list_sessions(self) -> list[Path]:
        """Return all active session.jsonl paths under this store root."""
        ...


class StoredSession:
    """Live handle to a session file on disk. Owns append and load; not the Session itself."""

    def __init__(self, path: Path) -> None: ...

    def append(self, event: AnyEvent) -> None:
        """Append a single event as a JSONL line. Rotates if the file exceeds 256KB."""
        ...

    def load(self) -> Session:
        """Replay all JSONL lines (across rotated files) into a Session."""
        ...

    def exists(self) -> bool:
        """Return True if the session file exists on disk."""
        ...


def rotate_if_needed(path: Path, max_bytes: int = 256 * 1024, keep: int = 3) -> None:
    """Rotate path → path.1 → … → path.N if path exceeds max_bytes."""
    ...
```

# tests/test_storage.py

def test_append_survives_crash(tmp_path):
    store = SessionStore(root=tmp_path)
    cwd = tmp_path / "project"
    cwd.mkdir()
    stored = StoredSession(store.session_path(cwd))
    session = new_session("claude-haiku-4-5-20251001", str(cwd))
    event = session.events[0]  # SessionStarted
    stored.append(event)
    # Simulate a crash: reload without the in-memory session
    reloaded = stored.load()
    assert len(reloaded.events) == 1


def test_rotation_creates_rotated_file(tmp_path):
    store = SessionStore(root=tmp_path)
    cwd = tmp_path / "project"
    cwd.mkdir()
    path = store.session_path(cwd)
    stored = StoredSession(path)
    # Write > 256KB worth of events
    session = new_session("claude-haiku-4-5-20251001", str(cwd))
    big_event = session.events[0]
    # Patch max_bytes to something small for the test
    for _ in range(100):
        stored.append(big_event, _max_bytes=100)  # test override
    assert path.with_suffix(".jsonl.1").exists() or path.stat().st_size > 0


def test_load_reads_across_rotated_files(tmp_path):
    path = tmp_path / "session.jsonl"
    stored = StoredSession(path)
    session = new_session("claude-haiku-4-5-20251001", str(tmp_path))
    stored.append(session.events[0])
    # Manually rotate
    rotate_if_needed(path, max_bytes=1)
    stored.append(session.events[0])
    reloaded = stored.load()
    assert len(reloaded.events) == 2


def test_workspace_isolation(tmp_path):
    store = SessionStore(root=tmp_path)
    dir_a = tmp_path / "project_a"
    dir_b = tmp_path / "project_b"
    dir_a.mkdir()
    dir_b.mkdir()
    assert store.session_path(dir_a) != store.session_path(dir_b)


def test_truncated_last_line_is_skipped(tmp_path):
    path = tmp_path / "session.jsonl"
    stored = StoredSession(path)
    session = new_session("claude-haiku-4-5-20251001", str(tmp_path))
    stored.append(session.events[0])
    # Corrupt the file: append a partial line
    path.open("a").write('{"incomplete":')
    reloaded = stored.load()
    assert len(reloaded.events) == 1  # partial line silently skipped

## Layer 2 — `config.py`: Game Configuration with `chz`

**Depends on:** nothing in this project. No imports from `session.py` or `storage.py`.

### Why `chz`

`chz` provides:
- **Immutable configuration objects** — config is frozen at construction; nothing mutates it mid-game.
- **`@chz.chz` classes** — like frozen dataclasses but keyword-only and designed for configuration.
- **`chz.nested_entrypoint`** — turns any function taking a config object into a CLI tool with free argument parsing (`realtalk game.model=claude-opus-4-6 display.no_color=true`).
- **`chz.replace()`** — create a modified copy of an immutable config for testing or overrides.
- **`@chz.init_property`** — computed fields that are evaluated at construction time.

`pydantic` handles input validation and coercion at the deserialization boundary (loading JSON config files). `chz` handles the in-process config object model. They are complementary: pydantic validates raw input; chz owns the live config.

### Configuration class hierarchy

```
RuntimeConfig
├── game: GameConfig
├── contributor: ContributorConfig
├── display: DisplayConfig
└── hooks: HookConfig
```

### `config.py` design

```python
from __future__ import annotations
import os
from enum import StrEnum
from pathlib import Path
import chz
from pydantic import BaseModel, field_validator


# ── Enums ────────────────────────────────────────────────────────────────────

class PermissionMode(StrEnum):
    READ_ONLY       = "read-only"
    WORKSPACE_WRITE = "workspace-write"
    FULL_ACCESS     = "full-access"
    PROMPT          = "prompt"


# ── Pydantic input models (deserialization + validation boundary) ─────────────
# Used only when loading JSON from disk. Convert to chz objects immediately after.

class RawGameConfig(BaseModel):
    model: str = "claude-haiku-4-5-20251001"
    min_turns_to_win: int = 8
    turn_hard_cap: int = 25
    arc_trigger_threshold: int = 80      # mood ≥ this → Invitation Arc
    mood_start_min: int = 30
    mood_start_max: int = 50
    security_start_min: int = 40
    security_start_max: int = 60
    reaction_delta_low: int = 3          # intensity 1 → ±3 pts
    reaction_delta_medium: int = 7       # intensity 2 → ±7 pts
    reaction_delta_high: int = 12        # intensity 3 → ±12 pts

    @field_validator("arc_trigger_threshold")
    @classmethod
    def arc_threshold_in_range(cls, v: int) -> int:
        if not (0 < v <= 100):
            raise ValueError("arc_trigger_threshold must be 1–100")
        return v


class RawContributorConfig(BaseModel):
    enabled: bool = False
    session_dir: str = "~/.realtalk/sessions"


class RawDisplayConfig(BaseModel):
    no_color: bool = False
    debug: bool = False


class RawHookConfig(BaseModel):
    pre_tool_use: list[str] = []
    post_tool_use: list[str] = []
    post_tool_use_failure: list[str] = []


class RawRuntimeConfig(BaseModel):
    game: RawGameConfig = RawGameConfig()
    contributor: RawContributorConfig = RawContributorConfig()
    display: RawDisplayConfig = RawDisplayConfig()
    hooks: RawHookConfig = RawHookConfig()


# ── chz configuration objects (live, immutable, used throughout the game) ────

@chz.chz
class GameConfig:
    model: str = "claude-haiku-4-5-20251001"
    min_turns_to_win: int = 8
    turn_hard_cap: int = 25
    arc_trigger_threshold: int = 80
    mood_start_min: int = 30
    mood_start_max: int = 50
    security_start_min: int = 40
    security_start_max: int = 60
    reaction_delta_low: int = 3
    reaction_delta_medium: int = 7
    reaction_delta_high: int = 12

    def reaction_delta(self, intensity: int) -> int:
        """Return the mood point delta for a player reaction of the given intensity (1–3)."""
        return {1: self.reaction_delta_low,
                2: self.reaction_delta_medium,
                3: self.reaction_delta_high}[intensity]


@chz.chz
class ContributorConfig:
    enabled: bool = False
    session_dir: str = "~/.realtalk/sessions"

    @chz.init_property
    def resolved_session_dir(self) -> Path:
        return Path(self.session_dir).expanduser()


@chz.chz
class DisplayConfig:
    no_color: bool = False
    debug: bool = False


@chz.chz
class HookConfig:
    pre_tool_use: list[str] = chz.field(default_factory=list)
    post_tool_use: list[str] = chz.field(default_factory=list)
    post_tool_use_failure: list[str] = chz.field(default_factory=list)


@chz.chz
class RuntimeConfig:
    game: GameConfig = chz.field(default_factory=GameConfig)
    contributor: ContributorConfig = chz.field(default_factory=ContributorConfig)
    display: DisplayConfig = chz.field(default_factory=DisplayConfig)
    hooks: HookConfig = chz.field(default_factory=HookConfig)

    @chz.init_property
    def api_key(self) -> str:
        key = os.environ.get("ANTHROPIC_API_KEY", "")
        if not key:
            raise EnvironmentError(
                "ANTHROPIC_API_KEY is not set.\n"
                "  export ANTHROPIC_API_KEY=sk-ant-..."
            )
        return key
```

### Three-tier config merge

Config is loaded in three tiers, lower tiers (more specific) win on conflict.
This mirrors the git config model: `system < global < local`.

```
Tier 1 (lowest): ~/.realtalk/config.json       ← user defaults
Tier 2:          .realtalk.json                 ← project settings (commit this)
Tier 3 (highest): .realtalk/settings.local.json ← machine overrides (gitignore this)
```

**Deep merge semantics:** dict keys are merged recursively. A project config that sets
`hooks.post_tool_use` does not erase a user-level `hooks.pre_tool_use` entry.

```python
class ConfigLoader:
    """Load and merge the three-tier configuration into a RuntimeConfig."""

    def __init__(self, cwd: Path = Path.cwd()) -> None:
        self.cwd = cwd

    def load(self) -> RuntimeConfig:
        raw = self._load_raw()
        validated = RawRuntimeConfig(**raw)   # pydantic validates/coerces
        return self._to_chz(validated)        # construct immutable chz objects

    def _load_raw(self) -> dict:
        tiers = [
            Path.home() / ".realtalk" / "config.json",
            self.cwd / ".realtalk.json",
            self.cwd / ".realtalk" / "settings.local.json",
        ]
        merged: dict = {}
        for path in tiers:
            if path.exists():
                import json
                data = json.loads(path.read_text())
                _deep_merge(merged, data)
        return merged

    def _to_chz(self, raw: RawRuntimeConfig) -> RuntimeConfig:
        return RuntimeConfig(
            game=GameConfig(**raw.game.model_dump()),
            contributor=ContributorConfig(**raw.contributor.model_dump()),
            display=DisplayConfig(**raw.display.model_dump()),
            hooks=HookConfig(**raw.hooks.model_dump()),
        )


def _deep_merge(base: dict, override: dict) -> dict:
    """Mutate base in place, merging override values. Dicts are merged recursively."""
    for key, value in override.items():
        if key in base and isinstance(base[key], dict) and isinstance(value, dict):
            _deep_merge(base[key], value)
        else:
            base[key] = value
    return base
```

### CLI integration via `chz.nested_entrypoint`

The `realtalk` CLI entry point uses `chz.nested_entrypoint` so every config field is
overridable from the command line without explicit `argparse` boilerplate:

```python
# realtalk/cli.py (preview — built in v0.3)
import chz
from realtalk.config import RuntimeConfig

def main(config: RuntimeConfig) -> None:
    ...

if __name__ == "__main__":
    chz.nested_entrypoint(main)
```

This allows:
```bash
realtalk game.model=claude-opus-4-6 display.no_color=true
realtalk contributor.enabled=true
```

CLI flags override the merged file config. The precedence order becomes:
```
user config < project config < local config < CLI flags
```

### `chz.replace()` for testing

Tests that need a modified config do not re-invoke `ConfigLoader`. They use `chz.replace`:

```python
def test_with_custom_arc_threshold():
    config = RuntimeConfig()
    config = chz.replace(config, game=chz.replace(config.game, arc_trigger_threshold=90))
    assert config.game.arc_trigger_threshold == 90
```
